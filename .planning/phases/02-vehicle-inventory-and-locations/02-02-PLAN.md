    Install Graph SDK: `cd api && npm install @microsoft/microsoft-graph-client`

    1. Create `api/src/services/vehicleService.ts` (from RESEARCH.md Pattern 2):
       - `getVehicles(filters: IVehicleFilters)`: SELECT with dynamic WHERE clause (isArchived=0, optional locationId/status/categoryId/search). JOIN Categories and Locations for names. All params via `request.input()`. ORDER BY make, model.
       - `getVehicleById(id: number)`: SELECT single vehicle by ID with JOIN for names. Return null if not found or archived.
       - `createVehicle(vehicle: VehicleInput, createdBy: string)`: INSERT with OUTPUT INSERTED.id. Set status to 'Available'. Include createdBy.
       - `updateVehicle(id: number, vehicle: VehicleInput, updatedBy: string)`: UPDATE all editable fields. Set updatedAt to GETUTCDATE(). Include updatedBy. Return boolean success.
       - `archiveVehicle(id: number, updatedBy: string)`: UPDATE set isArchived=1, archivedAt=GETUTCDATE(), updatedBy. Return boolean success. This is the soft delete per locked decision.
       - `updateVehicleStatus(id: number, status: string, updatedBy: string)`: UPDATE status column and updatedAt. Return boolean success. Do NOT allow status change on archived vehicles.
       - All functions use `getPool()` from database.ts and parameterized queries with `sql.NVarChar`, `sql.Int` types.

    2. Create `api/src/services/categoryService.ts`:
       - `getCategories()`: SELECT all categories WHERE isActive=1 ORDER BY name.
       - `createCategory(input)`: INSERT with OUTPUT INSERTED.id.
       - `updateCategory(id, input)`: UPDATE name and description. Return boolean.
       - `archiveCategory(id)`: SET isActive=0. Return boolean.

    3. Create `api/src/functions/vehicles.ts` (from RESEARCH.md code example):
       - Register 6 HTTP endpoints using `app.http()`:
         - `GET backoffice/vehicles` (listVehicles): requireRole('Admin','SuperAdmin'). Parse query params for filters. If user.effectiveRole === 'Admin', force locationId filter by querying the user's officeLocation. For now, use a helper that returns the location ID from the user's officeLocation (query Locations table by name matching user's officeLocation property -- this will be populated once we have the Graph-based officeLocation lookup; for now, accept a `locationId` query param override that Admin users can use, and add a TODO comment for the officeLocation auto-detection).
         - `GET backoffice/vehicles/{id}` (getVehicle): requireRole('Admin','SuperAdmin'). Return 404 if not found.
         - `POST backoffice/vehicles` (addVehicle): requireRole('Admin','SuperAdmin'). Validate body with VehicleInputSchema. Return 201 with id.
         - `PUT backoffice/vehicles/{id}` (editVehicle): requireRole('Admin','SuperAdmin'). Validate body with VehicleInputSchema. Return 200 or 404.
         - `DELETE backoffice/vehicles/{id}` (removeVehicle): requireRole('Admin','SuperAdmin'). Calls archiveVehicle (soft delete per locked decision). Return 200 or 404.
         - `PATCH backoffice/vehicles/{id}/status` (changeVehicleStatus): requireRole('Admin','SuperAdmin'). Validate body with VehicleStatusSchema. Return 200 or 404.
       - All handlers: get user via getUserFromRequest, check auth, return 401/403 as needed.
       - Use try/catch with 500 error response on unexpected errors.

    4. Create `api/src/functions/categories.ts`:
       - Register 4 HTTP endpoints:
         - `GET backoffice/categories` (listCategories): requireRole('Admin','SuperAdmin').
         - `POST backoffice/categories` (addCategory): requireRole('Admin','SuperAdmin'). Validate with CategoryInputSchema.
         - `PUT backoffice/categories/{id}` (editCategory): requireRole('Admin','SuperAdmin'). Validate with CategoryInputSchema.
         - `DELETE backoffice/categories/{id}` (removeCategory): requireRole('Admin','SuperAdmin'). Soft delete (isActive=0).

    1. Create `api/src/services/graphService.ts` (from RESEARCH.md Pattern 3):
       - Use `@microsoft/microsoft-graph-client` Client and `@azure/identity` DefaultAzureCredential.
       - `getGraphClient()`: Create Graph client using DefaultAzureCredential token for 'https://graph.microsoft.com/.default'. For local dev when AZURE_CLIENT_SECRET is set, use ClientSecretCredential instead of DefaultAzureCredential (requires AZURE_TENANT_ID, AZURE_CLIENT_ID, AZURE_CLIENT_SECRET env vars).
       - `getDistinctOfficeLocations()`: Paginate through `/users?$select=officeLocation&$top=999`. Collect all non-null, non-empty officeLocation values into a Set. Trim whitespace. Return sorted array of unique strings.
       - Handle Graph API errors gracefully -- log and throw a descriptive error.

    2. Create `api/src/services/locationService.ts`:
       - `getLocationsWithVehicleCounts()`: SELECT from Locations with LEFT JOIN and COUNT of non-archived vehicles per location. ORDER BY name. Return ILocation[] with vehicleCount populated.
       - `syncLocations(officeLocations: string[])`: Implement sync logic:
         a. Get all existing location names from DB.
         b. For each officeLocation not in DB: INSERT new Location (isActive=1, lastSyncedAt=now).
         c. For each DB location not in officeLocations: SET isActive=0 (deactivated -- location disappeared from Entra ID per locked decision about flagging for reassignment).
         d. For each DB location that IS in officeLocations AND was inactive: SET isActive=1 (reactivated).
         e. Update lastSyncedAt for all matched locations.
         f. Return ILocationSyncResult { added, deactivated, total }.

    3. Create `api/src/functions/locations.ts` (from RESEARCH.md location function example):
       - Register 2 HTTP endpoints:
         - `GET backoffice/locations` (listLocations): requireRole('Admin','SuperAdmin'). Return locations with vehicle counts.
         - `POST backoffice/locations/sync` (syncLocationsTrigger): requireRole('SuperAdmin') only (per locked decision -- sync is SuperAdmin-only). Call getDistinctOfficeLocations(), then syncLocations(). Return sync result (added, deactivated, total).
       - Error handling: try/catch with 500 response including error message for sync failures.

    4. Update `api/src/index.ts`: Add imports for all 3 new function modules:
