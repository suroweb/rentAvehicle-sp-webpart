**AllBookings page (components/AllBookings/AllBookings.tsx):**
Create a new admin page following the FleetManagement pattern (filter bar + DetailsList table + action dialogs). This is the primary admin booking management view.

Props: `apiService: ApiService`, `locations: ILocation[]`.

**Filter bar (top of page):**
- Location dropdown (Fluent UI Dropdown): all locations from props, plus "All Locations" option. Default: All.
- Status dropdown: options "All Statuses", "Confirmed", "Active", "Overdue", "Completed", "Cancelled". Default: All.
- Date range: two Fluent UI DatePicker components for "From" and "To" dates.
- Employee search: Fluent UI TextField with debounced input (300ms, follow VehicleFilterBar pattern) searching by employee display name.
- CommandBar or inline buttons: "Apply Filters" button to trigger fetch, "Clear Filters" to reset all.

**Table (Fluent UI DetailsList):**
Columns (use IColumn array with onColumnClick for sorting, following VehicleTable pattern):
1. ID (booking id, narrow)
2. Vehicle (make + model, e.g. "Toyota Corolla")
3. Employee (userDisplayName)
4. Location (locationName)
5. Start (formatted in location timezone using useTimezone hook)
6. End (formatted in location timezone)
7. Status (colored badge -- use StatusBadge component pattern or inline styles: Confirmed=blue, Active=green, Overdue=red, Completed=gray, Cancelled=gray-strikethrough)
8. Actions (Cancel button for Confirmed/Active/Overdue bookings only)

Sort: default by Start descending (newest first). Clickable column headers toggle asc/desc following VehicleTable pattern with `React.useMemo` for sorted items.

**Cancel dialog:**
When admin clicks Cancel button on a booking row:
1. Open a Fluent UI Dialog (not the simple ConfirmDialog -- this one needs a text field).
2. Dialog title: "Cancel Booking"
3. Dialog body: "Cancel booking #{id} for {vehicleName} by {employeeName}?" followed by a required TextField labeled "Reason for cancellation" with placeholder "Enter reason...".
4. Footer: "Cancel Booking" PrimaryButton (disabled when reason field is empty or whitespace-only) and "Close" DefaultButton.
5. On confirm: call `apiService.adminCancelBooking(bookingId, cancelReason)`. On success, close dialog and re-fetch bookings list. On error, show MessageBar in dialog.

**Data loading:**
- On mount and on filter apply: call `apiService.getAllBookings(filters)` with current filter values.
- Show Fluent UI Spinner during loading.
- Show "No bookings found" empty state when result is empty.

**Pagination:** Use DetailsList's built-in rendering. For v1, load all results (admin view typically has manageable row counts). No infinite scroll needed.

**IMPORTANT ES5 constraints:** All code must follow established patterns: function keyword, string concatenation, indexed for loops, no template literals, no arrow functions in array method callbacks.

**AllBookings styles (components/AllBookings/AllBookings.module.scss):**
- Filter bar: flexbox row with gap between filter controls, wrap on mobile.
- Table container: full width, overflow-x auto for mobile.
- Status badges: colored inline spans (reuse StatusBadge colors if available, or define: confirmed=#0078d4, active=#107c10, overdue=#d13438, completed=#a19f9d, cancelled=#605e5c).
- Cancel reason dialog: TextField with min-height for comfortable typing.
- Empty state: centered text with padding.

**AppShell routing (components/AppShell/AppShell.tsx):**
1. Add a new nav item for admin: key 'allBookings', text 'All Bookings', icon 'CalendarSettings' (or 'Bookmarks'), visible for Admin/SuperAdmin roles.
2. Add renderPage case for 'allBookings': render `<AllBookings apiService={apiService} locations={locations} />`.
3. Import AllBookings component.
4. The locations array is already loaded in AppShell (from Phase 2 FleetManagement pattern) -- pass it to AllBookings.
