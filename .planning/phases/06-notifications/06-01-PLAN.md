**3. Create `api/src/templates/adaptiveCards.ts`:**

Export preview text builder functions used in Teams activity feed notification `previewText` content. These are NOT full Adaptive Card JSON -- they are formatted preview strings for the activity feed:

- `buildBookingConfirmationPreview(vehicleName, startDate, endDate)` -> returns string like "Booking confirmed: Toyota Camry, Feb 26 10:00 - Feb 26 18:00"
- `buildManagerAlertPreview(employeeName, vehicleName, startDate, endDate)` -> returns string like "John Smith booked Toyota Camry, Feb 26 10:00 - Feb 26 18:00"
- `buildReminderPreview(type: 'pickup' | 'return', vehicleName, time)` -> returns string like "Pickup reminder: Toyota Camry at 10:00 AM" or "Return reminder: Toyota Camry due at 6:00 PM"
- `buildOverduePreview(vehicleName, returnTime)` -> returns string like "Overdue: Toyota Camry was due at 6:00 PM"

Format dates in a human-readable short format (e.g., "Feb 26, 10:00 AM"). Use Intl.DateTimeFormat or a simple formatter.

**4. Create `api/src/services/notificationService.ts`:**

Import `getGraphClient` from `./graphService.js`. Import template builders.

Functions to export:

a) `getManagerInfo(userId: string): Promise<{id: string, email: string, displayName: string} | null>`
   - Call `GET /users/${userId}/manager` with `$select=id,mail,displayName`
   - Return null on any error (404 = no manager, not an error)
   - Wrap in try/catch, log errors but never throw

b) `sendBookingConfirmationEmail(bookingId: number): Promise<void>`
   - Query booking with vehicle/location/category joins (same query pattern as calendarService `syncBookingToCalendars` for 'created' action)
   - Read `NOTIFICATION_SENDER_EMAIL` from env (required, log error if missing and return)
   - Read `APP_BASE_URL` from env
   - Build HTML using `buildConfirmationEmailHtml`
   - Call `POST /users/${senderEmail}/sendMail` via Graph client:
     ```
     { message: { subject, body: {contentType:'HTML', content}, toRecipients: [{emailAddress:{address}}] }, saveToSentItems: false }
     ```
   - Subject format: `"Booking Confirmed: ${make} ${model} (${licensePlate}) - ${locationName}"`
   - Wrap entire function in try/catch, log errors but never throw (fire-and-forget)

c) `sendTeamsActivityNotification(userId: string, activityType: string, previewText: string, bookingId: number): Promise<void>`
   - Read `APP_BASE_URL` from env
   - Call `POST /users/${userId}/teamwork/sendActivityNotification` via Graph client:
     ```
     { topic: { source: 'text', value: 'RentAVehicle', webUrl: `${appBaseUrl}?bookingId=${bookingId}` },
       activityType,
       previewText: { content: previewText },
       templateParameters: [] }
     ```
   - Wrap in try/catch. On error, log but don't throw. Activity feed may fail if Teams app not installed -- that's OK.

d) `notifyManagerOfBooking(bookingId: number, employeeUserId: string, employeeName: string, vehicleName: string, startDate: string, endDate: string): Promise<void>`
   - Call `getManagerInfo(employeeUserId)` -- if null, return silently
   - Build preview text using `buildManagerAlertPreview`
   - Call `sendTeamsActivityNotification(manager.id, 'managerBookingAlert', previewText, bookingId)`
   - Wrap in try/catch, log errors but never throw

e) `sendBookingNotifications(bookingId: number): Promise<void>`
   - This is the main orchestrator called from booking endpoints
   - Query booking details (same join query as sendBookingConfirmationEmail)
   - In parallel (Promise.allSettled):
     1. `sendBookingConfirmationEmail(bookingId)` -- email to employee
     2. `sendTeamsActivityNotification(userId, 'bookingConfirmed', confirmPreview, bookingId)` -- Teams notification to employee
     3. `notifyManagerOfBooking(bookingId, userId, displayName, vehicleName, startDate, endDate)` -- Teams notification to manager
   - Log any rejected promises but never throw
