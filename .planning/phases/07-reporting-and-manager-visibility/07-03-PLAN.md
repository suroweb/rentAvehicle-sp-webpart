**TeamBookings.tsx:**
Create a team bookings list page for managers.

- bookings: ITeamBooking[]
- loading: boolean
- error: string | null

- On mount (useEffect with empty deps): call apiService.getTeamBookings()
- Set loading=true before fetch, false after
- Catch errors and set error message

Layout:
1. **Page header:** "My Team's Bookings" heading with subtitle "Current and upcoming vehicle bookings for your direct reports"

2. **Loading state:** Fluent UI Spinner with "Loading team bookings..." label

3. **Error state:** Fluent UI MessageBar (error type) showing error message with a "Retry" action button

4. **Empty state:** MessageBar (info type) with "No active bookings found for your team members." This handles both "no direct reports" and "no bookings" cases cleanly.

5. **Bookings list:** Use Fluent UI DetailsList (consistent with AllBookings pattern) with these columns:
   - Employee (userDisplayName) -- managers CAN see names per locked decision
   - Vehicle (vehicleMake + ' ' + vehicleModel)
   - License Plate (vehicleLicensePlate)
   - Location (locationName)
   - Pickup (startTime formatted with Intl.DateTimeFormat using locationTimezone)
   - Return (endTime formatted with Intl.DateTimeFormat using locationTimezone)
   - Status (with color-coded StatusBadge: Confirmed=blue, Active=green, Overdue=red)

Use the same inline Intl.DateTimeFormat per-cell pattern as AllBookings.tsx (each booking may have a different location timezone).

Team view data scope (Claude's discretion): current + upcoming only (status IN Confirmed, Active, Overdue where endTime > now) -- this is already handled server-side by the API endpoint. Privacy-first: managers see operational oversight data, not historical tracking.

Sorting: sortable by column click (follow DetailsList pattern from AllBookings).
