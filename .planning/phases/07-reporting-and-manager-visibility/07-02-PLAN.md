**KpiCards.tsx:**
Create a KPI card row component with 4 cards:
- Utilization Rate (value + '%' suffix)
- Total Bookings (number)
- Active Bookings (number)
- Overdue (number)
Each card: styled div with large value text and label underneath.
Props: `{ kpi: IKpiSummary | null; loading: boolean }`.
Use CSS module classes from Reports.module.scss.
Show Fluent UI Spinner when loading. Show '--' when kpi is null.
Use function keyword (not arrow functions) for component declarations per project convention (ES5 compatibility).

**UtilizationChart.tsx:**
Create a utilization chart component using HorizontalBarChartWithAxis from @fluentui/react-charting.
Props: `{ data: IUtilizationData[] | IUtilizationVehicleData[]; isVehicleView: boolean; loading: boolean; selectedLocationName?: string }`.
- When showing location data: y-axis = location name, x-axis = utilization rate %
- When showing vehicle data: y-axis = "make model (plate)", x-axis = utilization rate %
- Use `getColorFromToken(DataVizPalette.color1)` for bar colors (per research pitfall #1 -- explicit colors avoid theming issues)
- Show "No data available" message when data array is empty
- Show Fluent UI Spinner when loading
- Show a heading: "Utilization by Location" or "Utilization at {selectedLocationName}" depending on view mode
- Bars should be clickable in location view to drill into vehicle detail (call onLocationClick prop)

**TrendChart.tsx:**
Create a trend chart component.
Props: `{ data: ITrendData[]; loading: boolean; granularity: 'daily' | 'weekly' }`.
Research suggests VerticalBarChart may support a line overlay via `lineLegendText`/`lineLegendColor` props. Try that first.
- If the combined chart works: render booking counts as bars, utilization % as line overlay
- If combined chart doesn't work: render VerticalBarChart for booking counts and a separate LineChart for utilization % stacked vertically
- Use DataVizPalette.color2 for bars, DataVizPalette.color3 for line (explicit colors)
- x-axis: period labels (dates). Format: 'MMM dd' for daily, 'Week of MMM dd' for weekly.
- Show "No trend data available" when empty
- Show Fluent UI Spinner when loading

**ReportExport.ts:**
Create CSV export utility (no React -- pure TypeScript):
- `escapeCSV(value: string | number | null | undefined): string` -- handle commas, quotes, newlines
- `downloadCSV(filename: string, headers: string[], rows: (string | number)[][]): void` -- BOM prefix for Excel UTF-8, Blob + URL.createObjectURL pattern, auto-click download link
- `exportSummaryCSV(data: IUtilizationData[]): void` -- columns: Location, Vehicle Count, Booking Hours, Available Hours, Utilization Rate (%). Filename: `fleet-utilization-summary-YYYY-MM-DD.csv`
- `exportRawDataCSV(data: IRawBookingRecord[]): void` -- columns: Booking ID, Vehicle (Make Model), License Plate, Location, Start Time, End Time, Duration (Hours), Status. NO employee data columns (privacy locked decision). Filename: `booking-records-YYYY-MM-DD.csv`

**Reports.tsx (main dashboard):**
Create the main dashboard layout component.
Props: `{ apiService: ApiService; locations: ILocation[]; categories: ICategory[] }`.

State:
- datePreset: DatePreset (default 'last30')
- dateRange: IDateRange (computed from preset via getDateRange utility from IReport.ts)
- selectedLocationId: number | undefined (for utilization drill-down and trend filter)
- selectedCategoryId: number | undefined (for trend filter)
- kpiData: IKpiSummary | null
- utilizationData: IUtilizationData[] | IUtilizationVehicleData[]
- isVehicleView: boolean (false = location view, true = vehicle detail)
- trendData: ITrendData[]
- loading states for each section

Layout (top to bottom):
1. **Header row:** "Fleet Reports" heading on left. Two CommandBarButtons on right: "Export Summary" and "Export Raw Data" (per locked decision -- two export buttons in top-right of dashboard header). Export Summary calls exportSummaryCSV. Export Raw Data fetches raw data from API then calls exportRawDataCSV.

2. **Date range preset row:** 4 DefaultButtons: "Last 7 Days", "Last 30 Days", "This Month", "This Quarter". Active preset gets primary style. Changing preset triggers data re-fetch.

3. **KPI cards row:** Render KpiCards component with kpi data.

4. **Utilization section:**
   - Header: "Utilization by Location" with a back button when in vehicle view
   - Location/category filter dropdowns (Fluent UI Dropdown) for trends -- location dropdown also controls utilization drill-down
   - UtilizationChart component
   - Clicking a location bar in location view: sets selectedLocationId, fetches vehicle-level utilization, toggles isVehicleView=true

5. **Trend section:**
   - Header: "Booking Trends"
   - TrendChart component
   - Below chart: small text summary of peak day/hour (Claude's discretion -- include it as a simple text insight if data allows, e.g., "Busiest day: Tuesday" derived from trend data)

Data fetching:
- On mount and whenever datePreset changes: fetch KPI, utilization (location level), and trends in parallel using Promise.all
- On location drill-down: fetch vehicle-level utilization for selected location
- On location/category filter change for trends: re-fetch trends with filters
- All fetches via apiService methods (defined in Task 2)
- Error handling: Fluent UI MessageBar for API errors, per-section error states
