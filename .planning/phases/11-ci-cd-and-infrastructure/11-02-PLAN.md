Target project structure:
```
infra/
├── main.bicep                    # Orchestrator

    ├── appServicePlan.bicep      # Consumption Y1

Naming convention (per research): `{prefix}-{resource-type}`
- App Service Plan: `{prefix}-plan`
- Function App: `{prefix}-func`
- SQL Server: `{prefix}-sql`
- Storage Account: `{prefix}st` (no hyphens — storage accounts require lowercase alphanumeric only)

SKU choices (per research):
- App Service Plan: Consumption Y1 ($0 base)
- SQL Database: Basic tier (~$5/mo)
- Storage Account: Standard LRS (~$0.01/mo)

Default region: `westeurope`

<task type="auto">
  <name>Task 1: Create Bicep resource modules</name>
  <files>infra/modules/appServicePlan.bicep, infra/modules/functionApp.bicep, infra/modules/sqlServer.bicep, infra/modules/storageAccount.bicep</files>
  <action>
Create the `infra/modules/` directory and four resource module files.

**1. `infra/modules/appServicePlan.bicep`**
- Resource: `Microsoft.Web/serverfarms@2022-03-01`
- Parameters: `name` (string), `location` (string)
- SKU: Y1 / Dynamic (Consumption plan for Azure Functions)
- Set `kind: 'functionapp'` and `properties.reserved: true` (required for Linux)
- Output: `id` (string) — the plan resource ID

**2. `infra/modules/storageAccount.bicep`**
- Resource: `Microsoft.Storage/storageAccounts@2022-09-01`
- Parameters: `name` (string), `location` (string)
- SKU: `Standard_LRS` (cheapest, sufficient for Function App state)
- Kind: `StorageV2`
- Output: `connectionString` (string) — construct from `listKeys()`:
  `'DefaultEndpointsProtocol=https;AccountName=${storageAccount.name};AccountKey=${storageAccount.listKeys().keys[0].value};EndpointSuffix=environment().suffixes.storage'`

**3. `infra/modules/functionApp.bicep`**
- Resource: `Microsoft.Web/sites@2022-09-01`
- Parameters:
  - `name` (string), `location` (string)
  - `appServicePlanId` (string)
  - `storageConnectionString` (string) — for AzureWebJobsStorage
  - `sqlServer` (string), `sqlDatabase` (string), `sqlPort` (string, default '1433'), `sqlUser` (string)
  - `@secure() sqlPassword` (string)
  - `tenantId` (string), `clientId` (string)
  - `@secure() azureClientSecret` (string)
  - `notificationSenderEmail` (string), `teamsAppId` (string), `appBaseUrl` (string)
  - `sharepointOrigin` (string)
- Kind: `'functionapp,linux'`
- Properties:
  - `reserved: true`
  - `serverFarmId: appServicePlanId`
  - `httpsOnly: true`
  - siteConfig:
    - `linuxFxVersion: 'node|22'`
    - `ftpsState: 'Disabled'`
    - `minTlsVersion: '1.2'`
    - CORS with `allowedOrigins: [sharepointOrigin]`, `supportCredentials: false`
    - appSettings array with ALL these entries:
      - `FUNCTIONS_EXTENSION_VERSION` = `'~4'`
      - `FUNCTIONS_WORKER_RUNTIME` = `'node'`
      - `WEBSITE_NODE_DEFAULT_VERSION` = `'~22'`
      - `AzureWebJobsStorage` = storageConnectionString
      - `AZURE_SQL_SERVER` = sqlServer
      - `AZURE_SQL_DATABASE` = sqlDatabase
      - `AZURE_SQL_PORT` = sqlPort
      - `AZURE_SQL_USER` = sqlUser
      - `AZURE_SQL_PASSWORD` = sqlPassword
      - `AZURE_TENANT_ID` = tenantId
      - `AZURE_CLIENT_ID` = clientId
      - `AZURE_CLIENT_SECRET` = azureClientSecret
      - `NOTIFICATION_SENDER_EMAIL` = notificationSenderEmail
      - `TEAMS_APP_ID` = teamsAppId
      - `APP_BASE_URL` = appBaseUrl
- Outputs: `name` (string), `defaultHostName` (string)

**4. `infra/modules/sqlServer.bicep`**
- Resources:
  - `Microsoft.Sql/servers@2022-05-01-preview` — SQL Server
  - `Microsoft.Sql/servers/firewallRules@2022-05-01-preview` — AllowAllWindowsAzureIps (start: 0.0.0.0, end: 0.0.0.0) — allows Azure services to connect
  - `Microsoft.Sql/servers/databases@2022-05-01-preview` — SQL Database
- Parameters: `serverName` (string), `databaseName` (string, default 'RentAVehicle'), `location` (string), `adminLogin` (string), `@secure() adminPassword` (string)
- SQL Server properties: `administratorLogin`, `administratorLoginPassword`, `publicNetworkAccess: 'Enabled'`
- Database SKU: `name: 'Basic'`, `tier: 'Basic'` (~$5/mo)
- Outputs: `serverFqdn` (string — `sqlServer.properties.fullyQualifiedDomainName`), `databaseName` (string)

IMPORTANT: Each module file should be self-contained with its own parameters and outputs. The main.bicep orchestrator will wire them together.

<task type="auto">
  <name>Task 2: Create main Bicep orchestrator, auth config module, and parameter file</name>
  <files>infra/main.bicep, infra/modules/authConfig.bicep, infra/main.bicepparam</files>
  <action>
Create the orchestrator, auth module, and parameter file.

**1. `infra/modules/authConfig.bicep`**
- Parameters: `functionAppName` (string), `entraClientId` (string), `entraTenantId` (string)
- Reference existing Function App: `resource functionApp 'Microsoft.Web/sites@2022-09-01' existing = { name: functionAppName }`
- Resource: `Microsoft.Web/sites/config@2022-09-01` with `name: 'authsettingsV2'`, parent: functionApp
- Properties:
  - `platform.enabled: true`
  - `globalValidation.requireAuthentication: true`
  - `globalValidation.unauthenticatedClientAction: 'Return401'` (API-only, no browser redirect)
  - `identityProviders.azureActiveDirectory`:
    - `enabled: true`
    - `registration.clientId: entraClientId`
    - `registration.openIdIssuer: 'https://sts.windows.net/${entraTenantId}/v2.0'`
    - `validation.allowedAudiences: ['api://${entraClientId}']`
  - `login.tokenStore.enabled: true`

**2. `infra/main.bicep`**
- Target scope: `targetScope = 'resourceGroup'` (default, but explicit for clarity)
- Parameters:
  - `location` (string, default 'westeurope')
  - `prefix` (string) — naming prefix for all resources
  - `sqlAdminLogin` (string) — SQL admin username
  - `sqlDatabase` (string, default 'RentAVehicle')
  - `sqlPort` (string, default '1433')
  - `@secure() sqlAdminPassword` (string)
  - `@secure() azureClientSecret` (string)
  - `entraClientId` (string) — existing app registration client ID
  - `entraTenantId` (string) — Azure tenant ID
  - `notificationSenderEmail` (string)
  - `teamsAppId` (string)
  - `appBaseUrl` (string)
  - `sharepointOrigin` (string) — e.g. 'https://contoso.sharepoint.com' (no trailing slash)
- Module calls (in dependency order):
  1. `module plan 'modules/appServicePlan.bicep'` — name: `'${prefix}-plan'`
  2. `module storage 'modules/storageAccount.bicep'` — name: `'${prefix}st'` (no hyphens)
  3. `module sql 'modules/sqlServer.bicep'` — serverName: `'${prefix}-sql'`, adminLogin: sqlAdminLogin, adminPassword: sqlAdminPassword
  4. `module func 'modules/functionApp.bicep'` — name: `'${prefix}-func'`, appServicePlanId: `plan.outputs.id`, storageConnectionString: `storage.outputs.connectionString`, SQL params from sql.outputs + parameters, auth params, app params, sharepointOrigin
  5. `module auth 'modules/authConfig.bicep'` — functionAppName: `func.outputs.name`, entraClientId, entraTenantId
- Outputs:
  - `functionAppUrl` (string) — `'https://${func.outputs.defaultHostName}'`
  - `sqlServerFqdn` (string) — `sql.outputs.serverFqdn`

**3. `infra/main.bicepparam`**
- Use `using './main.bicep'` syntax
- Non-secret defaults only:
  - `param location = 'westeurope'`
  - `param prefix = 'rentavehicle'`
  - `param sqlAdminLogin = 'sqladmin'`
  - `param sqlDatabase = 'RentAVehicle'`
  - `param sqlPort = '1433'`
- Add comments explaining that secrets are provided at deploy time:

  //     notificationSenderEmail='<value>' teamsAppId='<value>' \
  //     appBaseUrl='<value>' sharepointOrigin='<value>'
  ```

IMPORTANT:
- Bicep resolves dependencies automatically through parameter references. The auth module depends on func because it uses `func.outputs.name`.
- Do NOT include a health check exception from auth — no anonymous endpoints per user decision.
- The Storage Account is a hard requirement for Consumption plan Function Apps (AzureWebJobsStorage). Without it, the Function App will not start.
