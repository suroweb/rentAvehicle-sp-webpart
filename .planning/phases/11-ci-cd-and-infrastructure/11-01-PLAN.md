The API project has no ESLint configuration. Create one and install the required devDependencies.

1. Create `api/.eslintrc.json` with this exact content:
```json
{
  "root": true,
  "parser": "@typescript-eslint/parser",
  "parserOptions": {
    "project": "./tsconfig.json",
    "sourceType": "module"
  },
  "plugins": ["@typescript-eslint"],
  "extends": [
    "eslint:recommended",
    "plugin:@typescript-eslint/recommended"
  ],
  "rules": {
    "@typescript-eslint/no-explicit-any": "warn",
    "@typescript-eslint/no-unused-vars": ["warn", { "argsIgnorePattern": "^_" }],
    "@typescript-eslint/no-floating-promises": "error"
  },
  "ignorePatterns": ["dist/", "node_modules/"]
}
```

2. Install ESLint devDependencies in the api directory:
```bash
cd api && npm install --save-dev eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
```

3. Verify ESLint runs without configuration errors:
```bash
cd api && npx eslint src/ --max-warnings=999
```
The command should NOT fail with "No ESLint configuration found". It may report warnings — that is acceptable. It must not error on missing config.

IMPORTANT: Do NOT change any existing API source code. The ESLint config uses "warn" severity for most rules to avoid breaking CI on existing code. The one "error" rule (`no-floating-promises`) is critical for correctness.

<task type="auto">
  <name>Task 2: Create GitHub Actions CI workflow with parallel SPFx and API jobs</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create the CI workflow file at `.github/workflows/ci.yml`.

**Workflow triggers:** Push to main AND pull requests targeting main.

**Job 1: `build-spfx`**
- `runs-on: ubuntu-latest`
- `defaults.run.working-directory: spfx`
- Steps:
  1. `actions/checkout@v4`
  2. `actions/setup-node@v4` with `node-version: '22'`, `cache: 'npm'`, `cache-dependency-path: spfx/package-lock.json`
  3. `npm ci`
  4. **Generate CI env stub** — CRITICAL step. The SPFx `prebuild` script (`node tools/generate-env.js`) reads `../../.rentavehicle/secrets.json` which does not exist in CI and calls `process.exit(1)`. We must create a stub `src/config/env.generated.ts` BEFORE running heft. Use a `run` step:
     ```yaml
     - name: Generate CI env stub
       run: |
         cat > src/config/env.generated.ts << 'ENVEOF'
         // Generated by CI pipeline -- placeholder values for compilation only
         export const ENV = {
           AZURE_CLIENT_ID: 'CI_PLACEHOLDER',
           AZURE_TENANT_ID: '',
           APP_BASE_URL: '',
           DEV_USER_NAME: '',
           DEV_USER_EMAIL: '',
         } as const;
         ENVEOF
     ```
  5. `npx heft test --clean --production` — runs ESLint + TypeScript compilation + bundling. The `prebuild` npm script will run again but the file already exists so TypeScript resolves the import.
     WAIT — actually `npx heft` does NOT trigger npm lifecycle scripts like `prebuild`. Only `npm run build` does. So the stub just needs to exist for TypeScript to resolve the import. The heft commands will compile it directly.
  6. `npx heft package-solution --production` — creates the .sppkg
  7. Upload artifact (conditional on main branch):
     ```yaml
     - uses: actions/upload-artifact@v4
       if: github.ref == 'refs/heads/main'
       with:
         name: spfx-package
         path: spfx/sharepoint/solution/renta-vehicle.sppkg
     ```

**Job 2: `build-api`**
- `runs-on: ubuntu-latest`
- `defaults.run.working-directory: api`
- Steps:
  1. `actions/checkout@v4`
  2. `actions/setup-node@v4` with `node-version: '22'`, `cache: 'npm'`, `cache-dependency-path: api/package-lock.json`
  3. `npm ci`
  4. `npx tsc --noEmit` — type checking only, no output files needed for CI
  5. `npx eslint src/` — lint the API source using the config created in Task 1

Both jobs run in PARALLEL (no `needs` between them).

**Full workflow file structure:**
```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build-spfx:
    name: Build SPFx
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: spfx
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'
          cache-dependency-path: spfx/package-lock.json
      - run: npm ci
      - name: Generate CI env stub
        run: |
          cat > src/config/env.generated.ts << 'ENVEOF'
          // Generated by CI pipeline -- placeholder values for compilation only
          export const ENV = {
            AZURE_CLIENT_ID: 'CI_PLACEHOLDER',
            AZURE_TENANT_ID: '',
            APP_BASE_URL: '',
            DEV_USER_NAME: '',
            DEV_USER_EMAIL: '',
          } as const;
          ENVEOF
      - run: npx heft test --clean --production
      - run: npx heft package-solution --production
      - uses: actions/upload-artifact@v4
        if: github.ref == 'refs/heads/main'
        with:
          name: spfx-package
          path: spfx/sharepoint/solution/renta-vehicle.sppkg

  build-api:
    name: Build API
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: api
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'
          cache-dependency-path: api/package-lock.json
      - run: npm ci
      - run: npx tsc --noEmit
      - run: npx eslint src/
```

IMPORTANT: The two jobs have NO `needs` dependency — they run in parallel.
