**4. Create calendarService.ts (api/src/services/calendarService.ts):**

Create a new service file with the following functions:

**`escapeHtml(text: string): string`** -- Replace &, <, >, " with HTML entities.

**`buildVehicleEventBody(booking, vehicle, appBaseUrl): string`** -- HTML body for vehicle resource calendar per locked decisions:
  - Table with: Employee name, email, booking ID, pickup/return times
  - Vehicle details: make/model, plate, location, category
  - Link to booking in app

**`buildEmployeeEventBody(booking, vehicle, appBaseUrl): string`** -- HTML body for employee personal calendar per locked decisions:
  - Vehicle details: make/model, plate
  - Pickup location, return time
  - Deep link back to booking in app: `${appBaseUrl}?bookingId=${booking.id}`

**`buildStatusUpdateBody(status: string, originalBody: string, reason?: string): string`** -- Updates body to include status banner (CANCELLED / IN USE / RETURNED) at the top of the event body.

**`createCalendarEvent(userIdOrEmail: string, input: CalendarEventInput): Promise<string | null>`** -- Calls Graph API `POST /users/{userIdOrEmail}/events`. Returns the event ID on success, null on failure. Wraps in try/catch -- errors are logged but never thrown. CalendarEventInput type:
```typescript
interface CalendarEventInput {
  subject: string;
  bodyHtml: string;
  startDateTime: string; // ISO without Z suffix
  endDateTime: string;
  timeZone: string; // 'UTC'
  locationDisplayName: string;
  showAs: 'free' | 'busy';
  isReminderOn: boolean;
  reminderMinutesBeforeStart: number;
}
```

**`updateCalendarEvent(userIdOrEmail: string, eventId: string, updates: Partial<CalendarEventInput>): Promise<void>`** -- Calls Graph API `PATCH /users/{userIdOrEmail}/events/{eventId}`. Swallows errors with logging.

**`syncBookingToCalendars(bookingId: number, action: 'created' | 'cancelled' | 'checked_out' | 'checked_in', cancelReason?: string): Promise<void>`** -- The main orchestrator function. Steps:
1. Query the booking from DB (JOIN Vehicles for resourceMailboxEmail, make, model, licensePlate, locationName, categoryName; JOIN Locations for timezone).
2. Read `APP_BASE_URL` from `process.env.APP_BASE_URL` (fallback to empty string if not set).
3. Based on action:
   - **'created'**: Create TWO events:
     - Vehicle resource calendar: subject = `${userDisplayName} - ${make} ${model} (${licensePlate})`, showAs=busy, isReminderOn=false, location=locationName
     - Employee calendar: subject = `Vehicle Rental: ${make} ${model} (${licensePlate})`, showAs=free, isReminderOn=true, reminderMinutesBeforeStart=30, location=locationName
     - After creation, UPDATE Bookings SET vehicleCalendarEventId=X, employeeCalendarEventId=Y WHERE id=bookingId
   - **'cancelled'**: PATCH both events: prepend `[CANCELLED]` to subject, update body with cancellation banner + reason if provided. Do NOT change start/end times. Skip if event IDs are null.
   - **'checked_out'**: PATCH both events: prepend `[IN USE]` to subject, update body with check-out timestamp. Skip if event IDs are null.
   - **'checked_in'**: PATCH both events: prepend `[RETURNED]` to subject, update body with return timestamp. Skip if event IDs are null.
4. All errors caught and logged via `console.error` -- never thrown.

**Important notes:**
- Use `getGraphClient()` imported from `graphService.ts` for authentication.
- Start/end datetimes: strip the trailing 'Z' from ISO strings for Graph API (timeZone field handles UTC declaration).
- For the created action, the booking query must also SELECT v.resourceMailboxEmail. If resourceMailboxEmail is NULL, skip the vehicle calendar event creation but still create the employee event (vehicle may not have a mailbox provisioned yet).
- For update actions, first query the booking to get vehicleCalendarEventId, employeeCalendarEventId, and the resourceMailboxEmail from the joined vehicle. Only attempt updates where event IDs are not null.
