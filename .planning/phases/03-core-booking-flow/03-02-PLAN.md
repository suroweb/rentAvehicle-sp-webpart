**2. Create `spfx/src/webparts/rentaVehicle/hooks/useTimezone.ts`** — a React hook that accepts an IANA timezone string and returns memoized formatters:

```typescript
interface ITimezoneFormatters {
  formatDateTime: (utcDate: string | Date) => string;
  formatDateOnly: (utcDate: string | Date) => string;
  formatTimeOnly: (utcDate: string | Date) => string;
  timezoneAbbr: string;
  ianaTimezone: string;
}
```

Use `React.useMemo` keyed on the timezone string. Create `Intl.DateTimeFormat` instances with the `timeZone` option:
- `formatDateTime`: en-GB, year numeric, month short, day numeric, hour 2-digit, minute 2-digit, hour12 false
- `formatDateOnly`: en-GB, year numeric, month short, day numeric
- `formatTimeOnly`: en-GB, hour 2-digit, minute 2-digit, hour12 false
- `timezoneAbbr`: extract via `Intl.DateTimeFormat('en-US', { timeZone, timeZoneName: 'short' }).formatToParts()` finding the `timeZoneName` part

Also export a standalone utility function `localToUtcIso(year, month, day, hour, ianaTimezone): string` for converting user-selected local date+hour to UTC ISO string for the booking API. Use the `Intl.DateTimeFormat` with `timeZoneName: 'longOffset'` to determine the UTC offset, then compute UTC = local - offset. This is needed by BookingForm to convert the user's timezone-local selection to UTC for the API.
