```sql
CREATE TABLE Bookings (
  id INT IDENTITY(1,1) PRIMARY KEY,
  vehicleId INT NOT NULL REFERENCES Vehicles(id),
  userId NVARCHAR(255) NOT NULL,
  userEmail NVARCHAR(255) NOT NULL,
  userDisplayName NVARCHAR(255) NULL,
  startTime DATETIME2 NOT NULL,
  endTime DATETIME2 NOT NULL,
  status NVARCHAR(20) NOT NULL DEFAULT 'Confirmed'
    CHECK (status IN ('Confirmed', 'Active', 'Completed', 'Cancelled')),
  cancelledAt DATETIME2 NULL,
  cancelledBy NVARCHAR(255) NULL,
  createdAt DATETIME2 NOT NULL DEFAULT GETUTCDATE(),
  updatedAt DATETIME2 NOT NULL DEFAULT GETUTCDATE(),
  CONSTRAINT CK_Bookings_TimeOrder CHECK (endTime > startTime)
);
```

Add indexes:
```sql
CREATE INDEX IX_Bookings_VehicleId_Status ON Bookings(vehicleId, status) INCLUDE (startTime, endTime);
CREATE INDEX IX_Bookings_UserId ON Bookings(userId) INCLUDE (status, startTime, endTime);
CREATE INDEX IX_Bookings_StartTime ON Bookings(startTime) WHERE status IN ('Confirmed', 'Active');
```

**1. Create `api/src/services/bookingService.ts`** with these functions:

- `getAvailableVehicles(locationId: number, startTime: Date, endTime: Date, categoryId?: number): Promise<IAvailableVehicle[]>` — Query vehicles WHERE isArchived=0 AND status='Available' AND locationId=@locationId AND (optional categoryId filter) AND NOT EXISTS (SELECT 1 FROM Bookings b WHERE b.vehicleId = v.id AND b.status IN ('Confirmed', 'Active') AND b.startTime < @endTime AND b.endTime > @startTime). JOIN Categories and Locations (include l.timezone as locationTimezone). Use half-open interval overlap check (strict < and >, not <= and >=).

- `getVehicleDetail(vehicleId: number): Promise<IAvailableVehicle | null>` — Get single vehicle by ID with category/location joins (including l.timezone). Only return non-archived vehicles. This is the employee-facing version (no Admin role needed). Does NOT filter by status='Available' — employees should see the vehicle detail page even if it becomes unavailable (the booking form will fail gracefully).

- `getVehicleAvailability(vehicleId: number, days: number = 7): Promise<IVehicleAvailabilitySlot[]>` — Query confirmed/active bookings for vehicleId in the next N days from now. Return array of `{startTime, endTime, status}` representing booked slots. Used by the availability strip on the vehicle detail page.

- `createBooking(vehicleId: number, userId: string, userEmail: string, userDisplayName: string, startTime: Date, endTime: Date): Promise<{id: number} | {conflict: true}>` — **CRITICAL: Use SERIALIZABLE isolation level.** Create a new `sql.Transaction(pool)`, begin with `sql.ISOLATION_LEVEL.SERIALIZABLE`. First check: SELECT COUNT(*) as overlapCount FROM Bookings WHERE vehicleId=@vehicleId AND status IN ('Confirmed','Active') AND startTime < @endTime AND endTime > @startTime. If overlapCount > 0, rollback and return `{conflict: true}`. If 0, INSERT INTO Bookings with OUTPUT INSERTED.id, commit, return `{id}`. Catch errors — specifically check for SQL Server error 1205 (deadlock) and return `{conflict: true}` instead of throwing. Wrap rollback in try/catch to handle already-rolled-back transactions.

- `getMyBookings(userId: string): Promise<IBooking[]>` — Query bookings WHERE userId=@userId AND status != 'Cancelled'. JOIN Vehicles (make, model, year, licensePlate, categoryId, capacity, photoUrl), Categories (name), Locations (name, timezone). ORDER BY startTime DESC. The API response includes all joined fields so the frontend has everything it needs without additional calls.

- `cancelBooking(bookingId: number, userId: string): Promise<'cancelled' | 'not_found' | 'not_yours' | 'already_past'>` — First SELECT the booking by ID. If not found, return 'not_found'. If userId doesn't match, return 'not_yours'. If startTime <= current UTC time, return 'already_past' (can't cancel a booking that has started). UPDATE Bookings SET status='Cancelled', cancelledAt=GETUTCDATE(), cancelledBy=@userId WHERE id=@bookingId. Return 'cancelled'.

Follow existing service patterns from `vehicleService.ts`: import sql from 'mssql', import { getPool }, use `pool.request()` with `.input()` for all parameters.

**2. Create `api/src/functions/bookings.ts`** — register 6 employee-facing HTTP endpoints:

All endpoints: auth check via `getUserFromRequest`, return 401 if null. No role check needed — all authenticated users can use these endpoints (Employee minimum).

- `GET /api/vehicles/available` — route: `'vehicles/available'`. Parse query params: locationId (required, int), startTime (required, ISO string), endTime (required, ISO string), categoryId (optional, int). Validate locationId and times are present. Call `getAvailableVehicles()`. Return 200 with array.

- `GET /api/vehicles/{id}/detail` — route: `'vehicles/{id}/detail'`. Parse id from params. Call `getVehicleDetail()`. Return 404 if null, 200 with vehicle.

- `GET /api/vehicles/{id}/availability` — route: `'vehicles/{id}/availability'`. Parse id and optional `days` query param (default 7). Call `getVehicleAvailability()`. Return 200 with slots array.

- `POST /api/bookings` — route: `'bookings'`. Parse JSON body, validate with `BookingInputSchema.safeParse()`. Call `createBooking()` with user info from getUserFromRequest. If result has `conflict: true`, return **409** with `{error: 'This slot was just booked by someone else'}`. Otherwise return 201 with `{id}`.

- `GET /api/bookings/my` — route: `'bookings/my'`. Call `getMyBookings(user.userId)`. Return 200 with array.

- `DELETE /api/bookings/{id}` — route: `'bookings/{id}'`. Parse id. Call `cancelBooking(id, user.userId)`. Map results: 'cancelled' -> 200, 'not_found' -> 404, 'not_yours' -> 403, 'already_past' -> 400 with descriptive error message.

Register all 6 with `app.http()`. Use function names: `browseAvailableVehicles`, `getVehicleDetailEndpoint`, `getVehicleAvailabilityEndpoint`, `createBookingEndpoint`, `getMyBookingsEndpoint`, `cancelBookingEndpoint`.
