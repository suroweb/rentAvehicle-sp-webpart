<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Unified Date Range Picker across the entire booking flow:

1. **RangeCalendar Component** -- Inline single-month calendar grid in BookingForm with Airbnb-style range selection. Click start date (solid circle), click end date (solid circle), light band between. Two hour dropdowns below for start/end hours.

2. **Availability Strip Range Overlay** -- Semi-transparent colored band on the weekly strip showing the selected range. Click two free slots to set range. Drag handles on edges for fine-tuning. Snap-to-boundary prevents extending into booked slots. Range persists across week navigation.

3. **Day View Range Overlay** -- Range overlay on Day View hour columns. Same click-and-drag pattern as strip. Scope limited to single day (per user decision). Drag handles snap to booked boundaries.

4. **Browse Page Integration** -- VehicleBrowse uses same RangeCalendar component instead of separate DatePickers. Range passes to Detail page via IDateContext.

5. **Bidirectional Sync** -- Single source of truth (IRangeState in VehicleDetail). Any change in calendar, strip, day view, or hour dropdowns instantly updates all other surfaces.
  </what-built>
  <how-to-verify>
**Prerequisites:** Start the SPFx dev server: `cd spfx && gulp serve`

**Test 1: RangeCalendar in BookingForm (Detail Page)**
1. Navigate to any vehicle's detail page
2. In the right-side booking form, verify you see an inline calendar grid (not a popup DatePicker)
3. Click a date -- it should get a solid circle highlight (range start)
4. Click a different date -- it gets a solid circle and dates between get a light band
5. Verify the two hour dropdowns below the calendar show "Start time" and "End time"
6. Change start/end hours in dropdowns
7. Click "Review Booking" -- verify the review shows the correct range
8. Submit a booking -- verify it succeeds

**Test 2: Strip Range Overlay**
9. With a range selected, look at the Week View availability strip
10. Verify a semi-transparent blue overlay band spans from range start to range end
11. Navigate to a different week using arrow buttons -- verify the overlay shows the visible portion (or disappears if range is outside this week)
12. Navigate back to the original week -- verify overlay is still there (range persists)
13. Click two free (green) slots on the strip -- verify the range updates in the calendar and hour dropdowns
14. Look for drag handles at the start/end edges of the overlay
15. Drag a handle -- verify it adjusts the range. If it hits a booked (red) slot, verify it snaps to the boundary

**Test 3: Day View Range Overlay**
16. Switch to "Day View" tab
17. Verify a range overlay appears on the correct hour columns
18. Click two free hour slots -- verify range updates
19. If drag handles are visible, drag them -- verify snap-to-boundary on booked slots
20. Verify Day View changes are reflected in the calendar and hour dropdowns

**Test 4: Browse Page RangeCalendar**
21. Go back to the Browse page
22. Verify the filter bar shows an inline RangeCalendar instead of separate Start Date and End Date pickers
23. Select a date range on the calendar
24. Adjust start/end hours via dropdowns
25. Click Search -- verify results load correctly
26. Click a vehicle card to navigate to its Detail page
27. Verify the Detail page's range calendar and hour dropdowns show the same range you selected on Browse

**Test 5: Mobile (if available)**
28. Resize browser to mobile width (< 768px) or test in Teams mobile
29. Verify the mobile bottom bar shows a range summary (multi-day if applicable)
30. Open the bottom sheet -- verify RangeCalendar is visible inside
31. Day View mobile: verify hour slots show range highlighting

**Test 6: Edge Cases**
32. Same-day range: select start and end on same date -- verify single solid circle (start=end)
33. Past date protection: verify you cannot select dates before today
34. Booking conflict: book a slot that overlaps -- verify overlap warning and conflict suggestions still work
