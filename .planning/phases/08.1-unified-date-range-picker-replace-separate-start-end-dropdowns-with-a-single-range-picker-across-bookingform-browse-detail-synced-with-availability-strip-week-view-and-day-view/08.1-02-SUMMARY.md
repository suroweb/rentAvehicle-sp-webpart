**Interactive range overlay band on availability strip with two-click selection, drag handles for fine-tuning, and snap-to-boundary logic preventing selection into booked slots**

## Accomplishments
- Added semi-transparent blue overlay band on the strip that visually spans the selected range across days
- Implemented two-click range selection: first click sets start date/hour, second click sets end (with auto-swap if clicked before start)
- Added draggable handles at the start and end edges of the overlay for fine-tuning range hours
- Snap-to-boundary logic prevents dragging handles into booked or past time slots
- Range overlay correctly clamps to visible week -- partial visibility when range extends beyond, no overlay when range is entirely outside
- Bidirectional sync maintained: strip edits instantly propagate to calendar and hour dropdowns via shared IRangeState

## Decisions Made
- Used pointer events with setPointerCapture instead of mouse events for unified mouse + touch drag support
- requestAnimationFrame throttle on pointer move to prevent excessive state updates during drag (one update per frame)
- snapToBoundary walks backward for end-edge and forward for start-edge to find the nearest free slot boundary
- Range overlay rendered per-day-column inside .stripBlocks rather than spanning the whole grid -- simpler CSS positioning and correct behavior with flexbox layout
- computeDayOverlay is a pure function outside the component for clarity and potential future testing
- Changed AvailabilityStrip range/onRangeChange from optional to required since VehicleDetail always provides them (AvailabilityTimeline keeps optional until Plan 03)
