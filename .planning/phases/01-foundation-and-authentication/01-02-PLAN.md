**Models (`models/IUser.ts`):**

```typescript
export type AppRole = 'Admin' | 'Manager' | 'Employee';

export interface IUser {
  userId: string;
  displayName: string;
  email: string;
  role: AppRole;
}

export const ROLE_HIERARCHY: Record<AppRole, number> = {
  Employee: 0,
  Manager: 1,
  Admin: 2,
};

export function hasMinRole(userRole: AppRole, minRole: AppRole): boolean {
  return ROLE_HIERARCHY[userRole] >= ROLE_HIERARCHY[minRole];
}
```

**ApiService (`services/ApiService.ts`):**

Wraps `AadHttpClient` for all API calls:
- Constructor takes `AadHttpClient` instance and `baseUrl` (the Azure Functions app URL)
- `getMe(): Promise<IUser>` -- calls `GET {baseUrl}/api/me`, returns `IUser`
- `getHealth(): Promise<{ status: string; timestamp: string }>` -- calls `GET {baseUrl}/api/health`
- Private `get<T>(path: string): Promise<T>` -- generic GET helper using `this.client.get(url, AadHttpClient.configurations.v1)`, throws on non-OK responses
- All methods handle errors: throw descriptive errors for non-2xx responses

Note: Import `AadHttpClient` from `@microsoft/sp-http`. The API base URL should be passed as a constructor parameter (will come from webpart properties or a constant for now -- use a placeholder constant `API_BASE_URL` that can be swapped for the real URL later).

**AuthContext (`contexts/AuthContext.tsx`):**

React context that fetches and provides user identity:

1. `IAuthState` interface: `{ user: IUser | null; loading: boolean; error: string | null }`
2. `AuthContext = React.createContext<IAuthState>(...)`
3. `AuthProvider` component:
   - Props: `apiClient: AadHttpClient | null`, `userDisplayName: string`, `userEmail: string`, `children`
   - On mount, if `apiClient` exists, create `ApiService` instance and call `getMe()`
   - On success: set `user` from API response, `loading: false`
   - On error: set `error` with message, `loading: false`
   - If `apiClient` is null (init failed): set error "Failed to initialize API connection", `loading: false`
   - While loading: state is `{ user: null, loading: true, error: null }`
   - Provide fallback: if API call fails, create a minimal user from `userDisplayName` and `userEmail` with role `Employee` -- this way the shell still renders with a degraded experience
4. `useAuth()` custom hook: `React.useContext(AuthContext)` with null check
