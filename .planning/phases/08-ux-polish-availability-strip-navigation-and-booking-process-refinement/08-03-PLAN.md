**BottomSheet.tsx — Custom CSS bottom sheet component:**

Build a new component from scratch. Do NOT use Fluent UI Panel (it only supports left/right positioning, not bottom). Use CSS `position: fixed`, `transform: translateY()` transitions, and touch event handlers.

```typescript
interface IBottomSheetProps {
  isOpen: boolean;
  onDismiss: () => void;
  children: React.ReactNode;
}
```

Implementation details:
- **Overlay:** `position: fixed` covering entire viewport, `rgba(0,0,0,0.4)` background, `opacity` transition for fade in/out. Clicking overlay calls `onDismiss`.
- **Sheet:** `position: fixed`, `left: 0`, `right: 0`, `bottom: 0`, `border-radius: 12px 12px 0 0`, `max-height: 85vh`, `overflow-y: auto`, `z-index: 1001` (above Fluent UI layers per pitfall #3 in research).
- **Animation:** `transform: translateY(100%)` when closed, `translateY(0)` when open. Use `transition: transform 0.3s cubic-bezier(0.32, 0.72, 0, 1)` for native-feeling ease.
- **Drag handle:** A 36x4px rounded bar centered at top of sheet. Serves as swipe-to-dismiss target.
- **Swipe-to-dismiss:** On `touchStart` on the drag handle, record Y position. On `touchEnd`, if deltaY > 80px (downward), call `onDismiss`. Only attach swipe detection to the drag handle div (not the whole sheet) to avoid conflict with form scrolling (per pitfall #4 in research).
- **Safe area padding:** `padding-bottom: max(32px, env(safe-area-inset-bottom, 32px))` for iOS Teams mobile.
- Use JSX style (VehicleDetail and BookingForm already use JSX).

const touchStartRef = React.useRef<number>(0);

const handleTouchStart = React.useCallback(function onTouchStart(e: React.TouchEvent): void {
  touchStartRef.current = e.touches[0].clientX;
}, []);

const handleTouchEnd = React.useCallback(function onTouchEnd(e: React.TouchEvent): void {
  const deltaX = e.changedTouches[0].clientX - touchStartRef.current;
  if (Math.abs(deltaX) > 50) {
    if (deltaX < 0 && activeVehicleIndex < vehicles.length - 1) {
