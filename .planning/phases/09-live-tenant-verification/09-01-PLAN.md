  <how-to-verify>
**Step 1: Create Entra ID App Registration**
1. Go to Azure Portal -> Entra ID -> App registrations -> New registration
2. Name: "RentAVehicle" (or similar)
3. Supported account types: "Accounts in this organizational directory only"
4. Register

**Step 2: Create Client Secret**
1. In the app registration -> Certificates & secrets -> New client secret
2. Description: "dev-verification", Expires: 6 months
3. Copy the secret Value immediately (shown only once)

**Step 3: Grant Graph API Application Permissions**
1. In the app registration -> API permissions -> Add a permission -> Microsoft Graph -> Application permissions
2. Add these permissions:
   - `Calendars.ReadWrite` (create/update calendar events on any user/resource mailbox)
   - `Mail.Send` (send emails on behalf of any user)
   - `User.Read.All` (read user profiles and manager info)
   - `TeamsActivity.Send` (send Teams activity feed notifications)
3. Click "Grant admin consent for [tenant]" and confirm

**Step 4: Configure local.settings.json**
Update `api/local.settings.json` with the real tenant values:
```json
{
  "Values": {
    "AZURE_TENANT_ID": "<your-tenant-id>",
    "AZURE_CLIENT_ID": "<your-app-client-id>",
    "AZURE_CLIENT_SECRET": "<your-client-secret>",
    "NOTIFICATION_SENDER_EMAIL": "<licensed-mailbox-email>",
    "APP_BASE_URL": "https://yourtenant.sharepoint.com/sites/rentavehicle",
    "LOCAL_DEV_EMAIL": "<your-real-email>",
    "LOCAL_DEV_NAME": "<your-real-name>"
  }
}
```

**Step 5: Provision Resource Mailbox** (requires Exchange Online Management PowerShell)
```powershell
Connect-ExchangeOnline -UserPrincipalName admin@yourtenant.onmicrosoft.com
.\scripts\provision-vehicle-mailbox.ps1 -Make "Toyota" -Model "Camry" -LicensePlate "TEST001" -LocationName "YourOffice" -AppServiceAccount "your-app-service-account@yourtenant.onmicrosoft.com"
```
Copy the generated mailbox email from the script output.

**Step 6: Set Test User as Own Manager**
1. Go to Azure Portal -> Entra ID -> Users -> your test user
2. Edit Properties -> Manager -> Change manager
3. Search for and select the same user (self-manager)
4. Save

**Step 7: Verify API Builds and Starts**
```bash
cd api && npm run build && npm start
```
Confirm no authentication errors in the console output.

When all steps are complete, report back with:
- The AZURE_CLIENT_ID you created
- The resource mailbox email from the PowerShell script
- Confirmation that `npm start` shows no Graph auth errors
- Confirmation that the user's manager is set to themselves
