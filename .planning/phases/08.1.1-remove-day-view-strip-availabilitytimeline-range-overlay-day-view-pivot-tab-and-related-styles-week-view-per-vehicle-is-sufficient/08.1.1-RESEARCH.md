## Summary

Phase 08.1.1 is a pure deletion phase. Every artifact belonging to the Day View feature has been identified through codebase audit and must be removed. The scope is fully contained within two boundaries: the SPFx frontend (`spfx/src/`) and the Azure Functions API (`api/src/`). No new functionality is added — only existing code is deleted.

The deletion surface is larger than just two files. It spans six frontend files (two deletes, four edits) plus two API files (one delete, one edit). The most critical risk is TypeScript compilation breakage from dangling imports or references after deletion — the CONTEXT.md mandates a build verification step to catch this.

The AvailabilityTimeline component is cleanly isolated from AvailabilityStrip (Week View). They share no internal state or direct coupling beyond the shared `IRangeState` type from `RangeCalendar.tsx`. After removing the Pivot wrapper and AvailabilityTimeline, `AvailabilityStrip` renders directly with identical props it already receives.

**Primary recommendation:** Delete files first, then edit the files that reference them (VehicleDetail.tsx, ApiService.ts, IBooking.ts), then run TypeScript build to verify zero errors.


## Common Pitfalls

### Pitfall 1: Forgetting PivotItem in handleAvailabilityViewChange type signature
**What goes wrong:** The handler function `handleAvailabilityViewChange(item?: PivotItem)` references `PivotItem` as a type. If the Pivot import is removed but this handler function body is not, TypeScript will report `Cannot find name 'PivotItem'`.
**Why it happens:** Two separate removals are needed: the import line AND the handler function. Easy to miss.
**How to avoid:** Delete `handleAvailabilityViewChange` in the same edit as removing the Pivot import.
**Warning signs:** `TS2304: Cannot find name 'PivotItem'` during build.

### Pitfall 2: Stale ITimelineData in ApiService.ts import after method deletion
**What goes wrong:** Removing the `getTimeline()` method but leaving `ITimelineData` in the import line. TypeScript reports unused import (or worse, other linting errors).
**Why it happens:** The import line and method are on different lines — easy to remove one without the other.
**How to avoid:** Edit the import line and delete the method in the same file edit.
**Warning signs:** `'ITimelineData' is declared but its value is never read` (TS6133 warning or error depending on tsconfig).

### Pitfall 3: Missing the section heading requirement
**What goes wrong:** Removing the Pivot and AvailabilityTimeline without adding the "Availability" heading that CONTEXT.md says to keep.
**Why it happens:** The current JSX has NO heading for the availability area — only a JSX comment `{/* Availability views toggle */}`. The Pivot's own tab labels ("Week View") served as implicit section identification.
**How to avoid:** Add an `<h3>` or `<div>` heading "Availability" ABOVE the AvailabilityStrip in the replacement markup.
**Warning signs:** Post-removal UI has no section label above the strip.

### Pitfall 4: Incorrect spacing after Pivot removal
**What goes wrong:** The Pivot component had `className={styles.availabilityPivot}` with `margin-bottom: 24px`. After removal, if the wrapper div for AvailabilityStrip does not carry equivalent margin, the layout gap collapses.
**Why it happens:** CSS class was on the removed Pivot element, not inherited by its contents.
**How to avoid:** Add `.availabilitySection { margin-bottom: 24px }` to VehicleDetail.module.scss and apply it to the replacement wrapper div.
**Warning signs:** AvailabilityStrip visually collapses against the success MessageBar or BookingForm below it.

### Pitfall 5: Removing MyBookings.tsx Pivot (wrong target)
**What goes wrong:** A global grep for "Pivot" hits MyBookings.tsx, which uses Pivot for its own booking status tab system.
**Why it happens:** Pivot is used in two components: VehicleDetail (availability tabs — being removed) and MyBookings (booking status tabs — keep).
**How to avoid:** Only modify VehicleDetail.tsx for Pivot removal. MyBookings.tsx is completely out of scope.
**Warning signs:** MyBookings tab navigation stops working.
