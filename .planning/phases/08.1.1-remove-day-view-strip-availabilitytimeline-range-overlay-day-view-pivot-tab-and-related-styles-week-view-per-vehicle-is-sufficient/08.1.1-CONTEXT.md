# Phase 08.1.1: Remove Day View - Context

**Gathered:** 2026-02-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Remove the Day View feature entirely from VehicleDetail. Strip the AvailabilityTimeline component, its range overlay, the Day View pivot tab, and all related styles, state, interfaces, and API methods. Week View (AvailabilityStrip) per vehicle is the sole availability view going forward.

</domain>

<decisions>
## Implementation Decisions

### Pivot Tab Removal
- Remove the Pivot wrapper entirely — render AvailabilityStrip directly in VehicleDetail, no tabs
- Keep the "Availability" section heading above the strip for section clarity
- Preserve equivalent spacing (margin-bottom: 24px) where the Pivot was, applied directly to the AvailabilityStrip wrapper area
- Delete the `availabilityView` state and `handleAvailabilityViewChange` handler — no dead code

### File Deletion
- Delete AvailabilityTimeline.tsx entirely (100% Day View)
- Delete AvailabilityTimeline.module.scss entirely (100% Day View)
- Remove all imports of AvailabilityTimeline from other files (VehicleDetail.tsx)
- Remove `.availabilityPivot` CSS class from VehicleDetail.module.scss (Pivot no longer exists)

### API & Model Cleanup
- Delete `getTimeline()` method from ApiService.ts (only called by AvailabilityTimeline)
- Delete `ITimelineBooking` interface from IBooking.ts (Day View-only data model)
- Delete `ITimelineData` interface from IBooking.ts (Day View-only data model)

### Import & Reference Cleanup
- Remove all dead imports (AvailabilityTimeline, Pivot-related imports if no longer used)
- Full codebase sweep for string literals: 'day', 'Day View', 'dayView', 'day-view', 'AvailabilityTimeline' — catch any lurking references
- Hard remove everything — no fallback to week, the "day" concept no longer exists

### Build Verification
- Run build after removal to confirm zero TypeScript errors from broken imports/references
- Compilation must pass cleanly

### Claude's Discretion
- Exact approach to the spacing wrapper (inline style, CSS class, or existing utility)
- Order of file edits during removal
- Whether any Pivot-related Fluent UI imports become unused and should be cleaned up

</decisions>

<specifics>
## Specific Ideas

- "A perfect cleanup — less code and no day view"
- The removal is clean: AvailabilityTimeline and its styles are completely isolated from Week View (AvailabilityStrip)
- RangeCalendar and BookingForm are shared components — do NOT touch them

</specifics>
