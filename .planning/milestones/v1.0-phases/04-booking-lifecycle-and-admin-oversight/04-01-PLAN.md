**Schema changes (api/src/sql/schema.sql):**
Append ALTER TABLE statements to the end of the file (after existing Bookings CREATE TABLE):

1. Add three new nullable columns to Bookings table:
   - `checkedOutAt DATETIME2 NULL` -- timestamp when employee checked out the vehicle
   - `checkedInAt DATETIME2 NULL` -- timestamp when employee returned the vehicle
   - `cancelReason NVARCHAR(500) NULL` -- reason text when admin cancels a booking

2. Update the status CHECK constraint to include 'Overdue'. The existing inline CHECK constraint on the status column only allows `'Confirmed', 'Active', 'Completed', 'Cancelled'`. Must find and drop the existing constraint using `sys.check_constraints` and `sys.columns` system tables to get the auto-generated name, then recreate with 'Overdue' added. Use this pattern:
   ```sql
   -- Find and drop existing CHECK constraint on status column
   DECLARE @constraintName NVARCHAR(200);
   SELECT @constraintName = cc.name
   FROM sys.check_constraints cc
   INNER JOIN sys.columns c ON cc.parent_object_id = c.object_id AND cc.parent_column_id = c.column_id
   WHERE cc.parent_object_id = OBJECT_ID('Bookings') AND c.name = 'status';
   IF @constraintName IS NOT NULL
     EXEC('ALTER TABLE Bookings DROP CONSTRAINT ' + @constraintName);
   -- Add updated constraint
   ALTER TABLE Bookings ADD CONSTRAINT CK_Bookings_Status
     CHECK (status IN ('Confirmed', 'Active', 'Completed', 'Cancelled', 'Overdue'));
   ```

**Service functions (api/src/services/bookingService.ts):**

1. `checkOutBooking(bookingId: number, userId: string)` returning `'checked_out' | 'not_found' | 'not_yours' | 'too_early' | 'expired' | 'wrong_status'`:
   - Query booking by id. Return 'not_found' if missing, 'not_yours' if userId mismatch, 'wrong_status' if not Confirmed.
   - Calculate 30-min-before window: now must be >= startTime - 30 minutes. Return 'too_early' if before window.
   - Calculate 1-hour-after deadline: now must be <= startTime + 60 minutes. Return 'expired' if past.
   - Atomic UPDATE: SET status='Active', checkedOutAt=GETUTCDATE(), updatedAt=GETUTCDATE() WHERE id=@bookingId AND status='Confirmed'. Check rowsAffected -- if 0 return 'wrong_status' (race condition, already changed).

2. `checkInBooking(bookingId: number, userId: string)` returning `'checked_in' | 'not_found' | 'not_yours' | 'wrong_status'`:
   - Query booking by id. Return 'not_found' if missing, 'not_yours' if userId mismatch.
   - Atomic UPDATE: SET status='Completed', checkedInAt=GETUTCDATE(), updatedAt=GETUTCDATE() WHERE id=@bookingId AND status IN ('Active', 'Overdue'). Check rowsAffected -- if 0 return 'wrong_status'.

3. `getLocationTimeline(locationId: number, dayStart: Date, dayEnd: Date)` returning `{ vehicles: IAvailableVehicle[]; bookings: ITimelineSlot[] }`:
   - Query 1: All non-archived vehicles with status='Available' at locationId, JOINing Categories and Locations for display data. ORDER BY make, model.
   - Query 2: All bookings overlapping [dayStart, dayEnd) for vehicles at locationId, with status IN ('Confirmed', 'Active', 'Overdue'), JOINing Vehicles for display data. Include userId and userDisplayName. ORDER BY vehicle make/model, startTime.
   - Return both result sets.

4. `getBookingSuggestions(vehicleId: number, locationId: number, startTime: Date, endTime: Date, maxSuggestions: number = 3)` returning `IBookingSuggestion[]`:
   - Calculate duration = endTime - startTime.
   - Time shifts: iterate offsets [+1h, -1h, +2h, -2h, +3h, -3h, +4h, -4h]. For each, check if shifted slot has zero overlapping bookings (status IN Confirmed/Active/Overdue) for same vehicleId. Collect up to 2 time_shift suggestions.
   - Alt vehicles: query TOP 3 vehicles at same locationId (excluding original vehicleId), status='Available', isArchived=0, with no overlapping bookings for the original time range. Use NOT EXISTS subquery.
   - Combine, return up to maxSuggestions total.

5. `getAllBookings(filters: { locationId?: number; status?: string; dateFrom?: Date; dateTo?: Date; employeeSearch?: string })` returning `IBooking[]`:
   - Call autoExpireBookings() first.
   - Build dynamic query: SELECT from Bookings JOIN Vehicles JOIN Locations. Apply WHERE conditions based on which filters are provided. employeeSearch matches LIKE on userDisplayName. ORDER BY startTime DESC.
   - Return full booking records with vehicle name, location name, employee info.

6. `adminCancelBooking(bookingId: number, adminUserId: string, cancelReason: string)` returning `'cancelled' | 'not_found' | 'already_done'`:
   - Atomic UPDATE: SET status='Cancelled', cancelledAt=GETUTCDATE(), cancelledBy=@adminUserId, cancelReason=@cancelReason, updatedAt=GETUTCDATE() WHERE id=@bookingId AND status IN ('Confirmed', 'Active', 'Overdue').
   - If rowsAffected=0, check if booking exists at all: 'not_found' or 'already_done'.
