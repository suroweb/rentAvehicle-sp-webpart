**RangeCalendar.tsx -- New file:**

Create a reusable inline calendar component that wraps Fluent UI v8 `Calendar` with range selection. This component will be used in both BookingForm (detail page) and VehicleBrowse (browse page) in Plan 03.

1. **Define and export `IRangeState` interface** at the top of the file (this is the single source of truth contract used everywhere):
```typescript
export interface IRangeState {
  startDate: Date;
  startHour: number;
  endDate: Date;
  endHour: number;
}
```

2. **Define `IRangeCalendarProps`:**
```typescript
interface IRangeCalendarProps {
  range: IRangeState;

  minDate?: Date;

```

3. **Implement two-click range selection state machine:**
- Track `selectingEnd` boolean state (false = next click sets start, true = next click sets end)
- On `onSelectDate`:
  - If `selectingEnd` is false: set `startDate` to clicked date, set `endDate` to same date, flip `selectingEnd` to true
  - If `selectingEnd` is true: if clicked date is before `range.startDate`, swap (clicked becomes start, old start becomes end); otherwise set `endDate` to clicked date. Flip `selectingEnd` back to false
  - Special case: if user clicks the same date as startDate while selectingEnd=true, it becomes a single-day range (start=end=that date). Flip selectingEnd to false.

4. **Visual selection mode indicator:**
When `selectingEnd` is true, render a small text label below the calendar: "Click end date" in a subtle muted style. When false, show nothing (or optionally "Click start date" only if both dates are not yet set -- use Claude's discretion).

5. **Range styling via `customDayCellRef`:**
Use `calendarDayProps={{ customDayCellRef: ... }}` to apply CSS classes to day cells:
- Compare each cell's date against `range.startDate` and `range.endDate` (at midnight, ignoring time)
- Apply `rangeStart` class when date equals startDate
- Apply `rangeEnd` class when date equals endDate
- Apply `inRange` class when date is strictly between startDate and endDate
- Clear all three classes first on each call to prevent stale styles

**CRITICAL PITFALL (from research):** `customDayCellRef` may not re-fire for already-mounted cells when range state changes. To handle this, use a `key` prop on the Calendar that changes when range changes: `key={range.startDate.getTime() + '-' + range.endDate.getTime()}`. This forces Calendar re-mount on range change, ensuring `customDayCellRef` fires for all cells. This is the simplest reliable approach per research Pitfall 1.

6. **Render the Fluent UI Calendar:**
```typescript

  onSelectDate={handleSelectDate}

  isMonthPickerVisible={false}
  showGoToToday={false}
  minDate={minDate || getToday()}
  calendarDayProps={{ customDayCellRef: customDayCellRef }}
/>
```

Import `Calendar` and `DayOfWeek` from `@fluentui/react/lib/Calendar`. Do NOT import DatePicker.

7. **Helper function** `getToday()` returning today at midnight (same pattern as BookingForm).

```typescript
function getToday(): Date {
  const now = new Date();
  return new Date(now.getFullYear(), now.getMonth(), now.getDate());
}

function getNextFullHour(): number {
  const now = new Date();
  const nextHour = now.getHours() + 1;
  return nextHour >= 24 ? 0 : nextHour;
}

// Initialize range from browse page context or smart defaults
const [range, setRange] = React.useState<IRangeState>(function initRange(): IRangeState {
  const defaultStart = initialStartDate || getToday();
  const defaultStartHour = initialStartHour !== undefined ? initialStartHour : getNextFullHour();
  const defaultEnd = initialEndDate || defaultStart;
  const defaultEndHour = initialEndHour !== undefined
    ? initialEndHour
    : (defaultStartHour >= 23 ? 23 : defaultStartHour + 1);
  return {
    startDate: defaultStart,
    startHour: defaultStartHour,
    endDate: defaultEnd,
    endHour: defaultEndHour,
  };
});
```
