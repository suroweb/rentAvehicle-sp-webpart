## Decisions Made
- Used CSS Grid positioning for Day View range overlay (gridColumn spans hour range, gridRow spans all vehicles) -- natural fit with existing grid layout
- Pointer capture (setPointerCapture) for drag handles ensures dragging works even when pointer leaves the handle element
- Conservative snap-to-boundary approach uses booked hours from ALL vehicles on timeline (not just target vehicle) -- prevents any visual overlap
- VehicleBrowse filter bar restructured into vertical sections (dropdowns row, range calendar section, search button) for better readability with inline calendar
- AvailabilityTimeline props changed from optional to required since VehicleDetail (Plan 01) already passes range/onRangeChange
