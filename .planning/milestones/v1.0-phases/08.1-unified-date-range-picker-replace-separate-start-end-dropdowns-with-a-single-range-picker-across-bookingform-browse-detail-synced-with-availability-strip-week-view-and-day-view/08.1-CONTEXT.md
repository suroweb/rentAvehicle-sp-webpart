# Phase 08.1: Unified Date Range Picker - Context

**Gathered:** 2026-02-25
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the current 4 separate dropdowns (start date, start hour, end date, end hour) with a unified date range picker in the BookingForm. The range selection must be editable from three surfaces — the inline calendar in the form, the availability strip (week view), and the day view — all bidirectionally synced in real-time. Both the Browse page BookingForm and the Vehicle Detail side panel BookingForm use the same unified picker. Range state persists when navigating between Browse and Detail pages.

</domain>

<decisions>
## Implementation Decisions

### Range Picker Interaction
- Inline range calendar always visible in the form (not a popup)
- Single month grid with prev/next arrows for navigation
- Start and end dates get solid circles, dates in between get a light background band (highlighted band)
- Two hour dropdowns appear below the calendar for start hour and end hour
- User clicks start date on calendar, then clicks end date — band highlights the range

### Strip Range Display & Editing
- Selected range shown as a semi-transparent colored overlay band spanning from start date/hour to end date/hour
- Click a free slot to set start point, click another for end — then drag handles appear on edges to fine-tune
- When dragging/extending hits a booked slot, range snaps to the boundary (prevents invalid selections with visual feedback)
- Range spans across weeks — navigating to a different week shows the portion of the range visible in that week (or no overlay if range is outside that week). Range persists, never clears on week navigation

### Day View Hour Adjustment
- Same interaction pattern as strip: tap to place start/end hours, then drag handles to adjust
- Scope limited to single day only — day view edits hours for that day; multi-day range managed via calendar/form
- Same conflict behavior as strip: snap to boundary on booked slots
- Consistent behavior across strip and day view

### Sync Behavior
- Instant bidirectional sync — change range anywhere (calendar, strip, day view) and all other views update immediately
- One source of truth for the range state — no confirm/apply step
- Range preserved when navigating from Browse page to Vehicle Detail (extends existing Phase 8 date context wiring)
- Full bidirectional: drag on strip updates calendar, tap calendar updates strip, adjust day view updates both

### Claude's Discretion
- Whether day view shows a compact full-range header (e.g., "Feb 25 09:00 - Feb 27 17:00") above the hour slots
- Overlay band color and opacity values (should fit with Fluent UI and existing availability strip colors)
- Drag handle visual design (circles, bars, etc.)
- Loading/transition states during sync updates
- Mobile touch target sizing for drag handles

</decisions>

<specifics>
## Specific Ideas

- The interaction model should feel like a modern booking app (Airbnb-style date range selection on the calendar)
- Click-to-place then drag-to-adjust pattern should be consistent across strip and day view — same mental model everywhere
- Snap-to-boundary on conflicts gives clear feedback about why the range can't extend further
- The existing Phase 8 date context wiring (IDateContext interface, browse-to-detail prop passing) should be extended to carry the full range, not just a single date

</specifics>
