  <action>
    1. Install mssql and @types/mssql in the api project:
       `cd api && npm install mssql @azure/identity && npm install -D @types/mssql`

    2. Create `api/src/sql/schema.sql` with the full DDL (from RESEARCH.md Pattern 6):
       - `Locations` table: id (INT IDENTITY PK), name (NVARCHAR(128) UNIQUE), isActive (BIT DEFAULT 1), lastSyncedAt, createdAt, updatedAt
       - `Categories` table: id (INT IDENTITY PK), name (NVARCHAR(100) UNIQUE), description (NVARCHAR(500) NULL), isActive (BIT DEFAULT 1), createdAt, updatedAt
       - `Vehicles` table: id (INT IDENTITY PK), make, model, year, licensePlate (UNIQUE), locationId FK, categoryId FK, capacity, photoUrl, status with CHECK constraint ('Available','InMaintenance','Retired','Reserved'), isArchived (BIT DEFAULT 0), archivedAt, createdAt, updatedAt, createdBy, updatedBy
       - Indexes: IX_Vehicles_LocationId, IX_Vehicles_CategoryId, IX_Vehicles_Status (filtered WHERE isArchived=0), IX_Vehicles_IsArchived

    3. Create `api/src/services/database.ts` (from RESEARCH.md Pattern 1):
       - Singleton connection pool pattern. Export `getPool()` function.
       - Support two auth modes: when `AZURE_SQL_USER` env var is set, use SQL auth (local dev). Otherwise, use `azure-active-directory-default` auth type (production with Managed Identity).
       - Pool config: max 10, min 0, idleTimeoutMillis 30000.
       - encrypt: true, trustServerCertificate: true only when using SQL auth (local dev).

    4. Create `api/src/models/Vehicle.ts` (from RESEARCH.md zod schema example):
       - `VehicleInputSchema` (zod): make (string 1-100), model (string 1-100), year (int 1990 to currentYear+2), licensePlate (string 1-20), locationId (int positive), categoryId (int positive), capacity (int 1-50), photoUrl (string url max 500, nullable, optional).
       - `VehicleStatusSchema` (zod): status enum ['Available', 'InMaintenance', 'Retired'].
       - `VehicleInput` and `VehicleStatusInput` inferred types.
       - `IVehicle` interface: all DB columns including id, locationName, categoryName, isArchived, createdAt, updatedAt.
       - `IVehicleFilters` interface: locationId?, status?, categoryId?, search?, all optional.

    5. Create `api/src/models/Category.ts`:
       - `CategoryInputSchema` (zod): name (string 1-100), description (string max 500, nullable, optional).
       - `ICategory` interface: id, name, description, isActive, createdAt, updatedAt.

    6. Create `api/src/models/Location.ts`:
       - `ILocation` interface: id, name, isActive, vehicleCount (optional, for join queries), lastSyncedAt, createdAt, updatedAt.
       - `ILocationSyncResult` interface: added (number), deactivated (number), total (number).
  </action>

    - `grep -c "CREATE TABLE" api/src/sql/schema.sql` returns 3
    - `grep "getPool" api/src/services/database.ts` confirms the export
  </verify>
